# 真实模型简化交付

日期：2026-07-03

## 结论

本轮采用了“完整几何 + 分层计算代理”的双表示方案：

1. `sw/1`–`sw/5` 中的原始 STEP/STP 是唯一几何真源，未覆盖、未移动、未改名；
2. 顶层轻量接口家族用于快速检索；
3. 命中后展开到可逆的精确接口簇；
4. 最终位姿和实体碰撞仍回到原始 STEP 验证。

这样降低的是匹配计算模型的复杂度，不删除原模型中的孔、槽、实体、连接器或装配模块。

## 原模型保护结果

- 14/14 个源文件仍然存在；
- 14/14 个 SHA-256 与简化前一致；
- 14/14 个代理的原始特征成员覆盖率为 100%；
- SolidWorks 未参与代理生成，也未在后台运行。

每个精确簇都保存 `member_index_ranges`，可回到完整特征 JSON；每个顶层家族都保存 `child_cluster_index_ranges`，可展开到精确簇。因此简化层是分层索引，不是不可逆删除。

## 三层表示

| 层级 | 内容 | 用途 |
|---|---|---|
| L0 | 面积族、半径族和零件尺度摘要 | 混合池粗检索 |
| L1 | 严格容差下的共面、同轴精确簇 | 候选接口细化 |
| L2 | 全分辨率平面、圆柱面、孔特征 | 评分与位姿约束 |
| L3 | 原始 STEP/STP B-Rep | 精确碰撞和最终验证 |

L0 文件共 14 个，总大小约 1.23 MB。L1 可逆代理约 111.56 MB，只在命中后按需加载。

## 物理 STEP 合并试验及停止原因

曾尝试使用 OCCT `ShapeUpgrade_UnifySameDomain` 生成物理简化 STEP：

- 第 1 组每件仅 13 个面，同域合并后仍为 13 个面，没有有效降幅；
- 第 3 组风扇笼在 OCCT 7.9 `TKG2d.dll` 中触发 `0xc0000005`；
- 风扇模块能够运行到校验，但原 B-Rep 本身无效，写回后出现约 `1.29×10⁻⁶` 的相对体积差，被保护门拒绝。

因此没有把物理 STEP 合并作为正式方案，也没有用放宽门槛把失败包装成成功。相关原生入口已加显式风险确认开关，默认拒绝运行。

## Codex UI 崩溃定位

Windows 事件日志：

- 09:47:55：`python.exe` 在 `TKG2d.dll 7.9.3.0` 中以 `0xc0000005` 崩溃；
- 09:51:07：Codex app-server 同样以 `0xc0000005` 意外退出；
- 当时系统仍有约 17.9 GB 可用内存，因此不是内存耗尽。

Codex 扩展日志明确记录：

```text
Codex app-server process exited unexpectedly
code=3221225477 (0xc0000005)
```

后续不再从当前 Codex 会话运行该原生简化路径。正式代理生成是纯 Python JSON 运算，不加载 OCCT 或 SolidWorks。

## 当前边界

本轮完成了安全简化表示和复杂度验证，但尚未把 L0 → L1 → L2 的按需展开接入原 `pool_index.py`。因此不能把 98,262 直接宣称为新的端到端成功结果。

下一步应先实现分层候选召回审计，证明真实组所需接口不会在 L0/L1 被漏掉，再替换旧的全表面 `_prescreen`。
